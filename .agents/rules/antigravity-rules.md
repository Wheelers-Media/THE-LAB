Mobile-First UI/UX EngineeringTouch Targets: All buttons, interactive cards, and input elements must feature a minimum touch target area of $48 \times 48\text{px}$ to prevent misclicks on mobile devices.Layout Shifts: Utilize CSS Grid and Flexbox with explicit mobile-first breakpoints (sm:, md:, lg: in Tailwind). Default states must target standard mobile viewports ($370\text{px} - 430\text{px}$ width).Scroll Performance: Media elements must utilize smooth, hardware-accelerated scrolling (webkit-overflow-scrolling: touch). Avoid heavy animations that cause mobile frame drops.Visual Identity ConstraintsOLED Core Palette: Strict implementation of The Void (#000000), Midnight Carbon (#0D0D12), Signal White (#FFFFFF), Performance Blue (#0066FF), and Electric Cyan (#00E5FF).  Input Elements: All mobile forms and inputs must reject default browser styles. Use #000000 backgrounds with deep p-4 padding, rounded corners, and a micro-glow border (focus:border-labBlue) upon touch activation.  Asset Logic:logo.jpg must only render as pure white on absolute black backgrounds or on image overlays with a minimum of 60% black opacity.
